# Pixel-Prigel

**Digital Sticker Marketplace & Landing Page**  
*React · Firebase · Tailwind CSS · Cloudinary*

---

## Overview

Pixel-Prigel is a **portfolio project built with real product considerations**, designed as a landing page and lightweight marketplace for **digital (printable) sticker products**.

This project focuses on demonstrating **end-to-end fullstack development skills**, including frontend architecture, Firestore data modeling, authentication, and third-party service integration.

It is intentionally built beyond a tutorial scope and reflects **real-world decision making** commonly found in production applications.

---

## Project Goals

- Design a scalable structure for digital product management
- Implement clean and maintainable frontend architecture
- Build a realistic admin workflow with role-based access
- Demonstrate Firebase & Cloudinary integration in a real use case

---

## Core Features

### Product & Asset Management
- Full CRUD for digital sticker products
- SEO-friendly, slug-based product routing
- Support for:
  - Cover image
  - Gallery images
  - Downloadable digital assets
- Media storage and delivery handled by **Cloudinary**

---

### Authentication & Authorization
- Firebase Authentication
- Role-based access control:
  - Admin
  - Crew
- Protected admin routes and dashboard

---

### Firestore Data Architecture
- Schema designed specifically for **digital products**
- Read-optimized for landing page performance
- Structured for future extension (payments, access control)

---

### UI & UX
- Responsive and mobile-friendly layout
- Utility-first styling using **Tailwind CSS**
- Reusable components and custom hooks

---

## Tech Stack

### Frontend
- React.js
- React Router
- Tailwind CSS

### Backend & Services
- Firebase Authentication
- Cloud Firestore
- Cloudinary (media storage & optimization)

---

## Key Technical Decisions

- **Slug-based product identification**  
  Improves readability, SEO, and routing stability.

- **Cloudinary over default storage solutions**  
  Enables optimized delivery, transformations, and CDN support.

- **Firestore schema optimized for reads**  
  Suitable for landing pages and catalog-driven applications.

- **Tailwind CSS for UI development**  
  Ensures consistency, scalability, and faster iteration.
